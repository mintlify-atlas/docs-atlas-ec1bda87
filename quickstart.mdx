---
title: "Quickstart: install and run the trading bot"
description: "Install the Polymarket Copy Trading Bot and run your first arb or copy-trading session in under 10 minutes — from cloning to first live or dry-run trade."
---

This guide walks you through installing the bot, creating your `.env` configuration, and launching either the arb bot or the copy trader for the first time. By the end you will have a running bot connected to your Polymarket wallet on Polygon.

<Steps>
  <Step title="Prerequisites">
    Before you begin, make sure you have the following:

    - **Node.js 18 or higher** — check with `node --version`
    - **A Polymarket account** — sign up at [polymarket.com](https://polymarket.com) and connect a Polygon wallet
    - **A dedicated Polygon wallet** — use a wallet that holds only trading funds, not your main wallet
    - **USDC on Polygon** — the bot buys and sells USDC-denominated shares; a starting balance of $10–$20 is enough for testing
    - **A small amount of MATIC** — required for gas fees on Polygon; $1–$2 worth is sufficient
    - **A private RPC endpoint** — the default public endpoint is rate-limited; a free tier from [Alchemy](https://www.alchemy.com) or [Infura](https://www.infura.io) is strongly recommended

    <Note>
      Your **proxy wallet address** is different from your EOA address. After connecting your wallet on polymarket.com, find the proxy address in your account settings. You will need it in step 5.
    </Note>
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Parallax-Trading/polymarket-copy-trading-bot.git && cd polymarket-copy-trading-bot
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Create your .env file">
    Copy the example environment file to create your own configuration:

    <CodeGroup>

    ```bash Mac / Linux
    cp .env.example .env
    ```

    ```powershell Windows PowerShell
    Copy-Item .env.example .env
    ```

    </CodeGroup>
  </Step>

  <Step title="Fill in required settings">
    Open `.env` in your editor and fill in these three required variables:

    ```bash .env
    # Your Polygon EOA private key (controls the Polymarket proxy wallet).
    PRIVATE_KEY=0xYOUR_PRIVATE_KEY_HERE

    # The Polymarket proxy wallet address associated with your account.
    # Retrieve from https://polymarket.com after connecting your wallet.
    PROXY_WALLET=0xYOUR_PROXY_WALLET_HERE

    # Use a private endpoint (Alchemy / Infura / QuickNode) for reliability.
    POLYGON_RPC=https://polygon-mainnet.g.alchemy.com/v2/YOUR_KEY_HERE
    ```

    | Variable | What to put here |
    |---|---|
    | `PRIVATE_KEY` | The private key of your **dedicated trading wallet** (not your main wallet). Include the `0x` prefix. |
    | `PROXY_WALLET` | The proxy wallet address shown in your Polymarket account settings. This is **not** your EOA address. |
    | `POLYGON_RPC` | Your private RPC URL from Alchemy, Infura, or another provider. The default public endpoint can fail under load. |

    Leave `POLY_API_KEY`, `POLY_API_SECRET`, and `POLY_API_PASSPHRASE` blank — the bot derives and stores these automatically on first run.
  </Step>

  <Step title="Run the bot">
    Start the **arb bot** (BTC Up/Down arbitrage):

    ```bash
    npm start
    ```

    Or start the **copy trader** (mirrors target wallets):

    ```bash
    npm run copy
    ```

    The bot logs progress to the terminal and writes a persistent log to `bot.log` in the project directory.
  </Step>
</Steps>

<Tip>
  Run the copy trader in dry-run mode for your first session. Set `COPY_DRY_RUN=true` in your `.env` file to log all detected trades and sizing decisions without sending any actual orders. Verify the output looks correct before switching to live trading.
</Tip>
